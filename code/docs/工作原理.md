## PromptX 在中间干了什么

PromptX 通过 MCP 协议，给 AI 提供两类能力：

### 1. 角色（Role）

告诉 AI「你是谁」。

```
.promptx/resource/role/buffett/
├── buffett.role.md          # 主文件：人格 + 原则
├── thought/                 # 思维方式
│   ├── value-investing.thought.md
│   ├── margin-of-safety.thought.md
│   └── ...
├── knowledge/               # 专业知识
│   ├── buffett-letters.knowledge.md
│   └── ...
└── execution/               # 执行流程
    └── investment-workflow.execution.md
```

主文件 `buffett.role.md` 通过 `@!` 语法引用其他文件：

```markdown
<principle>
@!thought://value-investing
@!knowledge://buffett-letters
@!execution://investment-workflow
</principle>
```

### 2. 工具（Tool）

告诉 AI「你能干什么」。

```
.promptx/resource/tool/okx-btc/
└── okx-btc.tool.js
```

每个工具是一个 JS 文件，导出标准接口：

```javascript
module.exports = {
  getMetadata() { ... },    // 工具信息
  getSchema() { ... },      // 参数定义
  async execute(params) { ... }  // 执行逻辑
};
```

## 一次对话的完整流程

用户说「查一下 BTC 价格」：

```
1. 前端 WebSocket → AgentX
2. AgentX 带着巴菲特人设 → Claude API
3. Claude 决定调用工具，生成：
   {
     tool: "mcp__promptx__toolx",
     params: {
       yaml: "tool: tool://okx-btc\nparameters:\n  action: ticker"
     }
   }
4. AgentX → PromptX MCP Server
5. PromptX 执行 okx-btc.tool.js → 请求 OKX API
6. 返回价格数据 → Claude 组织回复
7. 流式返回 → 前端逐字显示
```

## Agent 怎么连接 PromptX

在 `src/agent.ts`：

```typescript
export const BuffettAgent = defineAgent({
  name: "Buffett",
  mcpServers: {
    promptx: {
      command: "promptx",      // 调用全局安装的 promptx
      args: ["mcp-server"]     // 启动 MCP 服务
    },
  },
  systemPrompt: BUFFETT_SYSTEM_PROMPT,
});
```

启动时，AgentX 会自动拉起 `promptx mcp-server`，它会：
1. 扫描 `.promptx/resource/` 目录
2. 加载所有角色和工具
3. 通过 MCP 协议暴露给 AI

## 为什么这样设计

**为什么用 PromptX 管理角色和工具？**

- 角色和工具**与代码分离**，改人设不用改代码
- 工具可以**热加载**，加新工具不用重启
- 标准化格式，**可复用**到其他项目

**为什么角色要拆成多个文件？**

- 单文件太长，维护困难
- 按「思维/知识/执行」分类，结构清晰
- 可以按需引用，灵活组合
