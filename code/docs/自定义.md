
# 自定义指南

改两类东西：**角色**（AI 是谁）、**工具**（AI 能干什么）。

都在 `.promptx/resource/` 目录下操作。

---

## 改角色

### 角色文件结构

```
.promptx/resource/role/buffett/
├── buffett.role.md              # 主文件 ⭐
├── thought/                     # 思维方式
│   ├── value-investing.thought.md
│   ├── margin-of-safety.thought.md
│   ├── circle-of-competence.thought.md
│   └── buffett-checklist.thought.md
├── knowledge/                   # 专业知识
│   ├── buffett-letters.knowledge.md
│   ├── classic-cases.knowledge.md
│   ├── tools-guide.knowledge.md
│   └── hk-stock-market.knowledge.md
└── execution/                   # 执行流程
    └── investment-workflow.execution.md
```

### 主文件格式

`buffett.role.md` 的结构：

```markdown
<role>
<personality>
# 角色名称

我是谁，我的背景，我的专长...

## 核心理念
...

## 能力范围
...

## 说话风格
...
</personality>

<principle>
@!thought://value-investing
@!thought://margin-of-safety
@!knowledge://buffett-letters
@!execution://investment-workflow
</principle>
</role>
```

`@!` 语法会引用同目录下的其他文件。

### 快速改人设

直接编辑 `buffett.role.md` 的 `<personality>` 部分。

比如想让 AI 更激进一点：

```markdown
## 核心理念

我相信价值投资，但也会适度参与趋势交易...
```

改完保存，重启服务生效。

### 创建新角色

```bash
# 1. 复制现有角色
cp -r .promptx/resource/role/buffett .promptx/resource/role/my-trader

# 2. 改名
mv .promptx/resource/role/my-trader/buffett.role.md \
   .promptx/resource/role/my-trader/my-trader.role.md

# 3. 编辑内容
```

然后在 `src/agent.ts` 里改 systemPrompt，告诉 AI 激活新角色：

```typescript
systemPrompt: `
请调用 mcp__promptx__action 激活 my-trader 角色。
激活后以该角色身份提供服务。
`
```

## 加工具

### 工具文件结构

```
.promptx/resource/tool/okx-btc/
└── okx-btc.tool.js     # 工具代码
```

一个目录，一个 JS 文件，名字保持一致。

### 工具代码模板

```javascript
/**
 * 工具名称
 */
module.exports = {
  // 工具信息
  getMetadata() {
    return {
      id: 'my-tool',
      name: '我的工具',
      description: '做某件事',
      version: '1.0.0'
    };
  },

  // 参数定义
  getSchema() {
    return {
      parameters: {
        type: 'object',
        properties: {
          action: {
            type: 'string',
            enum: ['query', 'submit'],
            description: '操作类型'
          },
          data: {
            type: 'string',
            description: '数据'
          }
        },
        required: ['action']
      }
    };
  },

  // 执行逻辑
  async execute(params) {
    const { action, data } = params;
    const { api } = this;  // PromptX 提供的 API

    if (action === 'query') {
      // 做查询...
      return { success: true, result: '查询结果' };
    }

    if (action === 'submit') {
      // 做提交...
      return { success: true, message: '已提交' };
    }

    return { error: '未知操作' };
  }
};
```

### 工具里可以用的 API

```javascript
async execute(params) {
  const { api } = this;

  // 导入 npm 包（会自动安装）
  const axios = await api.importx('axios');

  // 打日志
  api.logger.info('开始执行', { params });

  // 读环境变量
  const apiKey = process.env.MY_API_KEY;

  // ...
}
```

### 创建新工具示例

假设要加一个「发微信通知」的工具：

```bash
mkdir .promptx/resource/tool/wechat-notify
```

创建 `.promptx/resource/tool/wechat-notify/wechat-notify.tool.js`：

```javascript
module.exports = {
  getMetadata() {
    return {
      id: 'wechat-notify',
      name: '微信通知',
      description: '通过企业微信机器人发送通知',
      version: '1.0.0'
    };
  },

  getSchema() {
    return {
      parameters: {
        type: 'object',
        properties: {
          message: { type: 'string', description: '消息内容' }
        },
        required: ['message']
      },
      environment: {
        type: 'object',
        properties: {
          WECHAT_WEBHOOK: { type: 'string', description: '企业微信 Webhook URL' }
        }
      }
    };
  },

  async execute(params) {
    const { api } = this;
    const axios = await api.importx('axios');

    const webhook = process.env.WECHAT_WEBHOOK;
    if (!webhook) {
      return { error: '未配置 WECHAT_WEBHOOK' };
    }

    await axios.post(webhook, {
      msgtype: 'text',
      text: { content: params.message }
    });

    return { success: true, message: '已发送' };
  }
};
```

在 `.env` 里配置：

```env
WECHAT_WEBHOOK=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx
```

重启服务，对 AI 说「用微信通知发一条消息：测试」。

## 现有工具参考

### okx-btc

OKX 交易。

```yaml
# 查价格
tool: tool://okx-btc
parameters:
  action: ticker
  instId: BTC-USDT

# 查余额
tool: tool://okx-btc
parameters:
  action: balance

# 下单
tool: tool://okx-btc
parameters:
  action: trade
  instId: BTC-USDT
  side: buy
  ordType: market
  sz: "100"
```

### okx-analysis

技术分析。

```yaml
tool: tool://okx-analysis
parameters:
  action: signal
  instId: BTC-USDT
```

### futu-hk

富途港股模拟盘。

```yaml
tool: tool://futu-hk
parameters:
  action: quote
  symbol: "00700"  # 腾讯
```

### email-sender

发邮件（需配置 SMTP）。

```yaml
tool: tool://email-sender
parameters:
  action: send
  to: "xxx@example.com"
  subject: "标题"
  content: "<h1>内容</h1>"
```

## 部署

### Docker

```bash
docker build -t buffett-ai .
docker run -d -p 80:80 -p 5800:5800 \
  -e ANTHROPIC_API_KEY=你的key \
  buffett-ai
```

### Docker Compose

```bash
# 编辑 .env
docker-compose up -d
```
